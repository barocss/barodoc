ResponseField components help you document API response structures clearly.

## Basic Usage

<ResponseFieldGroup title="Response">
  <ResponseField name="id" type="string">
    Unique identifier for the created resource
  </ResponseField>
  <ResponseField name="created_at" type="string">
    ISO 8601 timestamp of creation
  </ResponseField>
  <ResponseField name="status" type="'active' | 'pending' | 'inactive'">
    Current status of the resource
  </ResponseField>
</ResponseFieldGroup>

## Complex Response

<ResponseFieldGroup title="User Response">
  <ResponseField name="data" type="object">
    The user object containing all user information
  </ResponseField>
  <ResponseField name="data.id" type="string">
    Unique user identifier
  </ResponseField>
  <ResponseField name="data.email" type="string">
    User's email address
  </ResponseField>
  <ResponseField name="data.profile" type="object">
    User's profile information
  </ResponseField>
  <ResponseField name="data.profile.name" type="string">
    Display name
  </ResponseField>
  <ResponseField name="data.profile.avatar_url" type="string | null">
    Profile image URL, null if not set
  </ResponseField>
  <ResponseField name="meta" type="object">
    Pagination and request metadata
  </ResponseField>
  <ResponseField name="meta.total" type="number">
    Total number of results
  </ResponseField>
</ResponseFieldGroup>

## Props

### ResponseFieldGroup
- `title` - Group heading
- `className` - Additional CSS classes

### ResponseField
- `name` (required) - Field name (supports dot notation)
- `type` - Field type
- `children` - Field description
